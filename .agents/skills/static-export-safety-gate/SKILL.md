# Static Export Safety Gate

## Workflow

1. Read `AGENTS.md`, `docs/architecture.md`, `docs/seo.md`, and `docs/deployment.md`.
2. Confirm the project remains Next.js 14, React 18, and `output: "export"` with `trailingSlash: true`.
3. Inspect changed files for forbidden runtime features: `app/api`, route handlers, server actions, ISR, runtime env reads, server-only APIs, dynamic image optimization, or `next start` as a production assumption.
4. For param routes, confirm `generateStaticParams` exists and params are handled with Next 14 synchronous patterns.
5. Run `npm run typecheck`; run `npm run build` for route, metadata, config, content-model, or deployment-facing changes.
6. Verify the artifact expectation: `out/` is generated by build, but must not be committed.

## Stop Conditions

Stop and report risk before editing if the request introduces a backend, API route, database, auth, payments, production secrets, runtime server, or new external runtime dependency.

## Output

Report static-export risks found, files checked, verification commands run, and any behavior that could not be verified locally.
